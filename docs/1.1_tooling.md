###   .gitignore
[.gitignore](../.gitignore) is a hidden files used by git when you stage change in the depot
  1. **Dependances**
     * `node_modules/`
        Installed dependencies
     * `.pnp.js`
        Dependances logic for Yarn Plug'n'Play 
     * `.pnp`
        Obtimisation binary for Yarn Plug'n'Play
  2. **Environnement**
     * `.env`
        Project environement variable
     * `.env.local`
        Local variables specific by machine
     * `.env.*.local`
        File type `.env.development.local` or `.env.production.local`
  3. **Docker secrets**
     * `secrets/`
        Docker compose secrets directory
  4. **Build outputs**
     * `dist/` & `build/`
        Post compilation directory for TypeScript -> JavaScript
     * `.next/`
        Next.js directory for build and cache
     * `out/`
        Output of static export
     * `.tmp` & `.cache`
        Temporary files automaticaly generated by tools or script
  5. **Logs**
     * `logs/`
        Logs directory
     * `*.log`
        Logs files
     * `npm-debug.log*`
        Debug files for npm
  6. **IDE**
     * `.vscode/`
        Local config for vscode
     * `.idea/`
        Config for JetBrains
     * `.swp`
        Temporary files for Vim
  7. **OS**
     * `.DS_Store`
        Hidden file generated by macOS
     * `Thumbs.db`
        File generated by Windows
  8. **Coverage**
     * `coverage/`
        Testing output

> This file is automaticly used by git

### package.json
[package.json](../package.json) is used to define the tool used by all the project
 * `"name": "maquis-devsecops-platform"`
    For npm and publication
 * `"version": "X.X.X`
    Actual version of the project (MAJOR.MINOR.PATCH)
 * `"private": true`
    Prevent involuntary publication on npm
 * `"scripts": {...}`
    Script list for npm or yarn, two types:
    Custom script `"build": "webpack"` run when call `npm run build`
    Lifecycle script have a specific name, they can be run like a custom script and automaticaly with `npm install` in the following order `preinstall` `install` `postinstall` `prepare` `precommit`
    * `{"prepare": "husky"}`
        Lifecycle script run the cmd `husky` when `npm install` or `npm run prepare` is run in terminal
 * `"lint-staged": {...}`
    Execute cmd on staged files
    * `{"**/*.{ts,tsx,js,json}": [...]}`
        Target files with targeted extention and apply the cmd in `[]`
        * `"prettier --write"`
        Automated formatting files by prettier
 * `"devDependencies": {...}`
    Developement dependencies
    * `"@commitlint/cli": "^X.X.X"` 
        CLI tool for commit checking
    * `"@commitlint/config-conventional": "^X.X.X"`
        Force the conventional commit format (feat: add login feature)
    * `"husky": "^X.X.X"`
        Tool for **hooks git** (pre commit)
    * `"lint-staged": "^X.X.X",`
        Scripts are run in modify files only (for optimisation)
    * `"prettier": "^X.X.X"`
        Auto formatting code tool

> When the depos is clone on a new PC the cmd `npm install` will use this file for installing all the needed tools listed. This file will create `.husky/` `node_modules/` and `package-lock.json`

###   commitlint
[commitlint.config.js](../commitlint.config.js) is the file that define the commit format used by the project. details [here](https://www.npmjs.com/package/@commitlint/config-conventional)
 * `extends: ['@commitlint/config-conventional']`
    Inerite of standard rules used by Javascript ecosystem (`feat(auth): add login page` or `feat: add login page`)
 * `rules: {...}`
    Apply a rule set
    * `'type-enum': [...]`
        Type chose is `type-enum` who need
        * `2` A num for hardening the error lvl detection
        * `'always'` Is when the rule is apply
        * Force the type of the commit to be one of the str contain in `[]`

> This file will be automaticaly used on every commit

###  prettier
[.prettierrc](../.prettierrc) is the config file of Prettier who re-style the code for make it more easly readable
 * `"semi": true`
    Add semi colon `;` at the end of every instruction
 * `"singleQuote": true`
    Replace double quote `"` by single quote `'`
 * `"tabWidth": 4`
    Indente whith 4 space
 * `"trailingComma": "all"`
    Will add a comma after the last element of object, parameters of fonction included
 * `"printWidth": 100`
    Width length can't be greater than 100 char, add automaticly a return to line
 * `"endOfLine": "lf"`
    Replace end of line type **CRLF** (Windows) by type **LF** (Unix/Mac)

> Modifie for change the general style of all code of the project

###   husky
[.husky](../.husky/) will store scripts to run automaticaly before and after a commit
 * [pre-commit](../.husky/pre-commit)
    Launch before a commit for garante a good quality of code
    * `npx lint-staged`
        Will launch the `lint-staged` tool in [package.json](../package.json) on files staged with `git add .` for exemple
 * [commit-msg](../.husky/commit-msg)
    Launch after a commit message for garante a good quality of history
    * `npx --no -- commitlint --edit $1`
        * `npx --no`
            Prevent from a new packet download, and unexpected execution
        * `commitlint`
            Tool from [commitlint.config.js](../commitlint.config.js)
        * `--edit $1`
            Check the text on path of temporary file of the new commit
